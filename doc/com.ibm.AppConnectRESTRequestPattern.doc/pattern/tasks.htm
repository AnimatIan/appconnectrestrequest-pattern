Tasks to complete after generating the App Connect REST Pattern

After generating a pattern instance, execute the following tasks:

  1. Open a web browser, log in to App Connect and from the Dashboard select New - Import Flow, and browse to the Template file which was generated by the pattern. This template will have a name of the format <SaaS Application>_<Verb>_<Object>.yaml like for example Dropbox_Create_folder.yaml
  2. Once imported, start the App Connect flow.
  3. Still in the App Connect web browser, locate the flow's Manage API panel:

    • Click Download API to download the Swagger document describing the App Connect REST API
    • Make a note of the Username
    • Make a note of the Password
  4. Import the Swagger document into your IIB Toolkit workspace and in the message flow, configure the App Connect REST Request node Definitions File property to refer to the Swagger file and select the Operation:

  5. Before deploying the IIB message flow, use the IIB runtime's commands to define the Basic Auth credentials to be used when IIB communicates with App Connect: 
    mqsisetdbparms <IntegrationNodeName> -n rest::appconuser01 -u <UsernameFromStep3> -p <PasswordFromStep3>
    mqsistop <IntegrationNodeName>
    mqsistart <IntegrationNodeName>
  6. Deploy the IIB message flow, and send in data to drive the flow, for example using the built-in IIB Flow Exerciser.
